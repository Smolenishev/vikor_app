Интерактивный веб-калькулятор для принятия обоснованных решений на основе математического метода **VIKOR** (VlseKriterijumska Optimizacija I Kompromisno Resenje). 

Инструмент разработан специально для задач логистики, ВЭД и управления цепями поставок (Supply Chain), где необходимо выбрать лучшую альтернативу из множества вариантов с противоречивыми критериями (например: цена vs скорость vs надежность).

## 🚀 Особенности и возможности

- ⚡ **Мгновенный расчет:** Математическое ядро на чистом JavaScript без тяжелых фреймворков.
- 💾 **Автосохранение (LocalStorage):** Все введенные данные, критерии и настройки сохраняются в браузере. Вы не потеряете работу при случайном обновлении страницы.
- 📱 **Полная адаптивность:** Удобная работа как на десктопе, так и на мобильных устройствах (Mobile-first подход).
- 📊 **Визуализация:** Построение интерактивных диаграмм индекса Q с помощью Chart.js.
- ✅ **Валидация по методологии:** Автоматическая проверка двух строгих условий компромисса (Приемлемое преимущество и Приемлемая устойчивость).
- ⚖️ **Умная нормализация:** Автоматический пересчет весов критериев, если их сумма не равна 1, с предупреждением пользователя.
- 📥 **Интеграция с Excel:** Наличие шаблона для офлайн-верификации расчетов.

## 🧮 Математическая основа

Метод VIKOR определяет компромиссное ранжирование, основанное на близости к идеальному решению. Калькулятор вычисляет три ключевых показателя для каждой альтернативы:

1. **$S_i$ (Групповая полезность):** Взвешенная сумма отклонений от идеала.
2. **$R_i$ (Индивидуальный риск):** Максимальное взвешенное отклонение по одному из критериев ("слабое звено").
3. **$Q_i$ (Индекс компромисса):** Итоговый показатель, рассчитываемый по формуле:

$$Q_i = v \cdot \frac{S_i - S^*}{S^- - S^*} + (1 - v) \cdot \frac{R_i - R^*}{R^- - R^*}$$

Где $v$ — вес стратегии принятия решений (по умолчанию 0.5, что обеспечивает баланс между максимизацией групповой полезности и минимизацией индивидуального риска).

## 🛠️ Как использовать

1. **Откройте файл** `index.html` в любом современном браузере (установка сервера не требуется).
2. **Настройте критерии (Блок 1):** 
   - Укажите названия критериев (например, "Стоимость", "Срок доставки").
   - Задайте вес каждого критерия (от 0 до 1).
   - Выберите тип: `Минимум` (чем меньше, тем лучше) или `Максимум` (чем больше, тем лучше).
3. **Добавьте альтернативы (Блок 2):** Введите названия вариантов (поставщиков, маршрутов) и их числовые значения по каждому критерию.
4. **Настройте стратегию (Блок 3):** При необходимости измените параметр $v$ (от 0 до 1).
5. **Нажмите "Рассчитать":** Получите ранжированный список вариантов, график и проверку условий компромисса.

> 💡 **Совет:** Чтобы начать с чистого листа, используйте кнопку 🗑️ "Сбросить все данные" в блоке альтернатив.

- **Разметка и стили:** HTML5, Tailwind CSS (подключен через CDN для нулевой конфигурации).
- **Логика:** Vanilla JavaScript (ES6+), без зависимостей от jQuery или React.
- **Визуализация:** Chart.js (для построения адаптивных графиков).
- **Хранение данных:** Web Storage API (`localStorage`).
